Astrogrid Development Processes

Introduction

The Astrogrid Development process is the result of wide consultation amongst developers and six iterations of experience of developing a complex, leading-edge system with a distributed team.

It exists both to ensure the quality of Astrogrid code and to maximize the efficiency of the team. Consequently, some aspects involve extra work for the individual in order to reduce the risk to the team as a whole.

The Development Process

  1. All development will be broken into discrete, small work units
  2. When work commences, a local snapshot of HEAD will be taken and a branch created for the work unit
  3. All development will take place on this branch
  4. Code will be committed to the branch very frequently (many times a day)
  5. Unit tests will be built/extended as code is developed (before code is developed, as per XP?)
  6. Code will be continuously unit tested
  7. When the work unit is complete and passes unit tests, it will be Integration Tested locally
  8. Problems within the work unit will be resolved
  9. A local copy of the latest HEAD code will be taken and the Integration Tests run locally again
  10. Again, any work unit problems will be addressed
  11. The developer is then left with true Integration issues which require the involvement of other engineers. These issues will be resolved on the branch
  12. Finally, the work unit branch will be committed to HEAD and the results of the automated builds will be monitored and issues resolved (potentially by repeating the steps above)

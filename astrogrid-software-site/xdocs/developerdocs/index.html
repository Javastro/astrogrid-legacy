Building AstroGrid from Source Code

The Java source code for AstroGrid may be obtained from our CVS repository by anonymous pserver access or via a web browser.

AstroGrid components are built using Maven and deployed to Tomcat or other J2EE-compliant application servers.

Development Environment

AstroGrid components must be buildable and executable on Windows and GNU/Linux platforms. Eclipse is the recommended IDE, but the choice is left to individual preference provided it does not impair our development processes. All code in CVS must be buildable by Maven at all times, both as snapshots and branched artifacts, and it is strongly recommended that developers install a copy of Maven and build using it regularly.

Development Process and Coding Standards

The AstroGrid Technical Lead has specified a development process. This process is the result of wide consultation with the AstroGrid developers and must be followed for the benefit of all developers. Don't dump work on your colleagues because you were too lazy to follow it.

Minimum coding standards have been agreed for AstroGrid code.

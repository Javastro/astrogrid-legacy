Minimum level of Javadoc

Astrogrid components should have the following minimum level of javadoc:

  • Every component should have an overview.html giving (surprise) an overview of the component.
  • Every package should have a package.html summarising the purpose of the package, and perhaps giving examples of use, or describing the main classes.
  • Every class should have a description.
  • Client-delegate classes should be have complete javadoc.

An example of good javadoc is the org.astrogrid.config package of the Common library.

Where appropriate use of maven's packagegroup feature in your project's project.xml to group together related packages. See MavenFAQ for how it's done. The CEA component demonstrates the benefits.

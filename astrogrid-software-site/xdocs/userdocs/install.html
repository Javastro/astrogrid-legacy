Installing Astrogrid

Installing and configuring individual Components

AstroGrid components are installed by deploying them to a J2EE-compliant container such as Tomcat. Users unfamiliar with this procedure should consult the Tomcat documentation. Individual components require specific configuration properties to be set. Please consult the AstroGrid Maven documentation for detailed instructions on configuring individual components.

Which components do I need to install?

If you have data you wish to publish on AstroGrid then you will be mostly interested in the Publishers AstroGrid Library (PAL). If you wish to plug your application software into the AstroGrid architecture, then see the Common Execution Architecture documentation. Finally, if you wish to host an AstroGrid web-site, manage a community of users, or manage users' storage facilities then you should see the Portal, Community and mySpace components.

Installing a typical AstroGrid using AGINAB

The quickest way to get started with Astrogrid is to install a complete set of components on a single machine using AGINAB (Astrogrid In A Box). AGINAB is a Maven script that downloads the appropriate Astrogrid components, installs them onto your local Tomcat server and configures them. Instructions for the use of AGINAB can be found here. Note that AGINAB is only appropriate for testing and demonstration purposes.

UType resolver

This is utype-resolver version @VERSION@

This is an implementation of the UType resolver described in Norman's IVOA Note (local copy).

The proposal in that Note is that UTypes should be identified by URIs, and that organisations should be able to create new UTypes fairly promiscuously, as long as those new UTypes' relationships with other more general, and better known, UTypes are described in a machine-readable fashion. This resolver makes it easy to use these relationships.

If an application were presented with some data labelled by a UType http://example.org/utype#new-utype, then it can immediately discover which better-known UTypes this is a subclass of, by dereferencing the URL http://hostport/utype-resolver/superclasses?http://example.org/utype%23new-utype, where the hostport is the location of this or any other UType resolver, and the # character has been replaced by the escape sequence %23. This returns a plain list of superclasses of that UType: if the application recognises any of these, it can process the labelled data as that type.

The resolver does not have to be preloaded with information about all the UTypes in the world, or even any of them. Since part of the proposal in the Note is that UType namespaces be dereferenceable, the resolver can simply retrieve the machine-processable representation of the UType when it is first asked about it, and cache it more-or-less permanently thereafter.

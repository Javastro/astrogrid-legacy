Quaestor: generic SPARQL endpoint

GET http://hostport/quaestor/kb/access-control?sparql=xxx
The string xxx is a URLEncoded SPARQL query which is made against the access-control knowledgebase.
GET http://hostport/quaestor/kb/access-control[/instances]

Retrieve either the complete access-control model, or the instances submodel. The format is obtained from the Accept header, which must contain one of the following MIME types:

  • application/n3 and text/rdf+n3 for n3.
  • application/rdf+xml for RDF/XML
  • text/plain for N-Triples
POST http://hostport/quaestor/kb/access-control?sparql
Takes a SPARQL query as the request body
PUT http://hostport/quaestor/kb/access-control
Create a new knowledgebase named `access-control'. The request content contains the metadata for the knowledgebase, which is at present just a string. It is an error if the knowledgebase already exists.
PUT http://hostport/quaestor/kb/access-control/instances
Upload the request body as the `instances' knowledgebase. The request may have a Content-Type header indicating the format of the body, which, if it is present, must be one of RDF/XML, RDF/XML-ABBREV, N-TRIPLE or N3, with the first one being the default.
PUT http://hostport/quaestor/code
Eval the request body as Scheme code. This is only possible if the server has been configured to permit it.

Quaestor: generic SPARQL endpoint

This is Quaestor @VERSION@, released @RELEASEDATE@

Quaestor is a generic triplestore and SPARQL endpoint, which allows you to upload ontology and instance data, and query the result through a RESTful interface. It's built on top of Jena.

The following files are available for download:

@WARFILE@
The Tomcat .war file. Drop this in a running Tomcat, or upload it to the management interface. The Quaestor server's interface is documented here, as well as on the front page of the running Quaestor service.
@JETTYZIP@
The standalone Jetty server. Unzip this, and start the server with the start-jetty.sh shell script. This presents the same interface as the Tomcat version above.
@CLIENTZIP@
API docs, and a small jar file which includes a REPL which allows you to talk to the server if you want to (I imagine rather few folk really want either, so this is I'm afraid barely documented)
@LIBJAR@
Library jar file, which allows you to create Quaestor-like applications fairly straightforwardly. Well, it would, if there were any documentation for that (coming soon, promise...).

The source code is in the AstroGrid project's CVS repository (check out directory experiments/norman/quaestor). You can also browse this with ViewVC. That should be publicly available -- get in touch with me if you have difficulty getting at it. The build instructions in the top-level README should at least help build it; again, get in touch if you have problems.

The script element is an activity that executes some inline scripting code.
It is a very versatile activity. The script element contains an optional description element;
a mandatory body element containing the script text, and a step-execution-record
which records the execution of the script. A script may reference workflow variables,
reading & storing data in them. The changes to the workflow variables are visible further
on in the workflow. A script may also define local variables, functions, etc.
These are only available to the script itself & are not visible to subsequent scripts or script expressions.
Hence any result that is to be accessed later should be stored in a previously-defined workflow variable.

in the variable 'a' store the value 'hello world' (java.lang.String): <set var="a" value="hello world" />
in the variable 'b', store the value '1' (java.lang.String): <set var="b" value="1" />
in the variable 'c', store the value 1 (java.lang.Integer): <set var="c" value="${1}" />
in the variable 'd' store the value '1 + 1' (java.lang.String): <set var="d" value="${c} + 1" />
in the variable 'e' store the value 2 (java.lang.Integer): <set var="e" value="${c + 1}" />
in the variable 'now' store the current date (java.lang.Date): <set var="now" value="${new java.util.Date()}" />
in the variable 'nSt' store current date (java.lang.String) - note trailing space: <set var="nSt value="${new java.util.Date()} " />
in the variable 'f' store the value '1 - hello world and goodbye.": <set var="f" value="${c} - ${a} and goodbye. "/>
in the variable 'u' store val 'www.astrogrid.org' (java.net.URL): <set var="u" value="${new java.net.URL('www.astrogrid.org')} />
in the variable 'scheme' store the value 'http' (java.lang.String): <set var="scheme" value="${u.getScheme()}" />

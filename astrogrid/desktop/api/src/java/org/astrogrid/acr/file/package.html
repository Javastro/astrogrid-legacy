Access to remote filesystems

This module provides uniform access to the following types of file system:

FilesystemExampleRead FilesWrite FilesList DirectoriesNotes
Local Diskfile:///Users/frog/Documents YYY
VOSpacevos://uk.ac.le.star!frog/dir/results.vot YYY
Myspaceivo://uk.ac.le.star/frog#/dir/results.vot YYY
User's workspaceworkspace:/dir/results.vot YYY Redirects to user's Myspace or VOSpace as appropriate
FTPftp://user:pass@server.com/dir/file.txt YYY To authenticate, supply username and password in URL
Secure FTPsftp://user:pass@server.com/dir/file.txt YYY To authenticate, supply username and password in URL
HTTPhttp://www.server.org/path/file.vot YNN Accepts query parameters after ?
Temporarytmp://dir/file.txt YYY All temporary files are deleted when AR terminates.

The full list of supported filesystems is available from the {@link org.astrogrid.acr.file.Systems} component. The other components in this module are

{@link org.astrogrid.acr.file.Manager}
Operations on files and folders: list, read, write, create, delete, move, copy
{@link org.astrogrid.acr.file.Info}
Access metadata about files and folders: size, content-type, modification date.
{@link org.astrogrid.acr.file.Name}
Construct, decompose and manipulate URI filenames

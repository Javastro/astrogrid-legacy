Simplest possible implementation of CEA

cea implementation that calls (static) methods on java classes. this implementation might be of some use in deployment, but is primarily for development / testing

This package could be used to provide standard 'helper applications' at each cea / jes server - e.g. the 'send mail' application still needs a bit of work -

  • reflecting parameter types - currently everything is called 'string'
  • should be able to export methods in more than one static class at the same time
  • should have a way of providing the description information not available via reflection alone (e.g. ucds, documentation) - possible methods
    • either reading a separate configuration file (blergh),
    • metadata in source (like bean descriptors)
      • ,
    • or attributes of the source file (nanning, jakarta attributes,xdoclet even)

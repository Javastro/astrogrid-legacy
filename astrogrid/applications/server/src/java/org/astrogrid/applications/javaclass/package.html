Simple implementation of a CEA Application Provider

This package defines a cea provider that calls (static) methods on java classes. This implementation might be of some use in deployment (for exporting libraries of helper functions), and is also used in development / testing. It also provides a good example of the structure of a CEA provider implementation.

Structure

{@link org.astrogrid.applications.javaclass.JavaClassApplicationDescriptionLibrary} constructs and contains a set of {@link org.astrogrid.applications.javaclass.JavaClassApplicationDescription} object for a parameter static class. {@link org.astrogrid.applications.javaclass.JavaClassApplication} executes an application by calling the appropriate method of the static class.

{@link org.astrogrid.applications.javaclass.JavaClassParameterDescription} extends the standard parameter description by recording the java type of the parameter - in turn {@link org.astrogrid.applications.javaclass.JavaClassParameterAdapter} extends the standard parameter adapter by converting the parameter string to the correct java type.

{@link org.astrogrid.applications.javaclass.SampleJavaClassApplications} is a sample set of java applications - hello world, sum two numbers, etc.

Further Work

This package could be used to provide standard 'helper applications' at each cea / jes server - e.g. the 'send mail' application. It could be extended by

  • being able to export methods in more than one static class at the same time - although this can be worked-around at the moment by defining an ApplicationDescrtiptionLibrary for each static class, and then combining using the CompositeApplicationDescriptionLibrary
  • providing a method of defining the description information not available via reflection alone (e.g. ucds, documentation) - possible methods
    • reading a separate configuration file (blergh),
    • metadata in source (like bean descriptors, componentDescriptor)
      • ,
    • attributes of the source file (nanning, jakarta attributes,xdoclet)(maven supports aspectj, aspectwerkz, but these provide a bit too much)

Classes to transform input parameters to and output results from the http service. This allows extra parameters to be added in, parameters split up, etc. The flexibility gives a number of potential benefits

    it allows us to use more descriptive parameter names
    it allows us to call services where the parameters aren't just passed in as ?key=value pairs at the end of the url. Sometimes 'parameters' make up part of the url itself.
Post-processing of the results might be required when an http service returns an html page for example, and the results need to be stripped out.

In general, SIAP services will not require this flexibility, therefore if no scripts are supplied in the registry, a default do-nothing transformation is used.

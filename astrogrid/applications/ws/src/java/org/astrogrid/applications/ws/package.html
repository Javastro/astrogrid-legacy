Contains specialized classes to make a CEA adapter for a general SOAP based web service

The general idea is that this component is an adapter can take a CEA tool pass it on to a general web service with normal asynchronous CEA semantics

This is not as simple as the WS hype might lead you to believe however, as there are severe interoperability problems. There are some limitations however;

  • At the moment the Web service that is to be called needs to have simple atomic types in the parameters
  • The web service needs to have been registered as a CEA application - there are tools aid this in the org.astrogrid.applications.ws.registry package
  • There is a one to one name correspondance between the CEA parameters and the web service message names
  • The call to the web service is made using the JAX-RPC DII interface which does lead to some interoperability problems with the "style" of web service (why did SOAP have to get so complex...? we are just passing an XML document after all...)

Configuration package; provides a simple set of methods for application-wide access to properties, using both traditional key/value pairs and XPath querying.

Background

The configuration package has to be able to:

  • Provide simple, backward-compatible access to key/value properties
  • Run without any configuration file present, with values set by the code, for unit tests
  • Load from several files (eg user-entered values, plus programmer-entered values)
  • Provide a getproperty(XPath) returning XML nodes (implies a load from XML) for structured properties

    Simple

    For the most simplest of configurations, use the static methods on the SimpleConfig class. These access a single (hidden) instance, and allows programmers to use the configuration with a minimum of code, like this:

    Initialisation: 

    
SimpleConfig.autoload();
    or 
    
SimpleConfig.load("SomeFile.cfg");
    Using Properties: 
    
String fruit = SimpleConfig.getProperty("GlobalFruit");

    Instance

    Create using the provided factory if this is really your thing :-) If you provide the factory method with a key object, you can reget the same one. For example: 
    
PropertyConfig config = ConfigFactory.getPropertyConfig("REGISTRY");
    returns the same instance as another same call elsewhere in the code: 
    
ConfigFactory.getPropertyConfig("REGISTRY");

    Loading

    You can request a load from urls or files. The autoload() method will look in JNDI and the system environment for urls, and the local directory for a default filename. You can load from as many files as you like for the PropertyConfig; only the latest will work at the moment for the XML version.

    Accessing XML

    XPath doesn't seem to have settled yet, so be prepared for this one to break occasionally. In the meantime, it takes the name of an element and does a search for all nodes matching that element name.

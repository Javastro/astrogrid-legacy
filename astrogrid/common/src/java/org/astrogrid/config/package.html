Configuration package; provides a simple set of methods for application-wide access to properties, using both traditional key/value pairs and XPath querying.

Background

The configuration package has to provide the following:

  • Simple, backwardly compatible access to simple key/value properties
  • Ability to load from several files (eg user-entered values, plus programmer-entered values)
  • Provide a getproperty(XPath) returning XML nodes (implies a load from XML)

    Simple

    For the most simplest of configurations, use the static methods on the SimpleConfig class. These access a single (hidden) instance, and allows programmers to use the configuration with a minimum of code.

    Instance

    Create using a factory if this is really your thing :-)

    Loading

    You can request a load from urls, files, and the autoload will look in JNDI and the system environment for urls, and the local directory for a default filename. You can load from as many files as you like for the PropertyConfig; only the latest will work at the moment for the XML version.

    Accessing XML

    XPath doesn't seem to have settled yet, so be prepared for this one to break occasionally.

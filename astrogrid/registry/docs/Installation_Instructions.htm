This is the Installation Instructions for the Registry which currently requires several manual jobs, but is expected to be easier in future iteration 05. AssumptionsK/i>

  • These instructions assume you have access to the CVS system.
  • You have an Apache Tomcat server installed.
  • You have an Apache Axis installed (For Server Side Install Only).
  • You have an Apache Cocoon installed (For Client Side Install Only, must have access though to axis classes ex: axis.jar file can be used in cocoon).
  • A jconfig.jar file is accessible for both Server side and Client side access.

Server Side
  • From CVS Grab the registry-server-4_0.jar file located in astrogrid/registry/Release/server cvs location.
  • Place the registry-server-4_0.jar file in the axis/WEB-INF/lib directory.
  • Be sure the jconfg.jar file is also in this axis/WEB-INF/lib directory.
  • Make a config file. A sample config file is in cvs under astrogrid/registry/src/config
  • The config file is now well documented please read it. It tells of cvs locations of where to find templates and example registry file plus the editing of a registry file which is required for new registries.
  • The example config file points to a more full Registry file, for setting up new registries please use and point in your config file to another registry file. An example sample registry is given in cvs at location astrogrid/registry/src/xml/ called new_registry_sample.xml. Be sure to read this xml file for setting up your registry.
  • Edit the tomcat/webapps/axis/WEB-INF/web.xml file. And an env-entry right before the "" closing tag:. An example entry is given below please change to accomodate to your config file directory. org.astrogrid.registry.config C:\astro_registry_server.xml java.lang.String
  • Start the tomcat server
  • deploy the registry wsdd files. Normal command is java org.apache.axis.client.AdminClient
  • astrogrid/registry/src/wsdd/adminquerydeploy.wsdd
  • astrogrid/registry/src/wsdd/querydeploy.wsdd
  • Server side is complete

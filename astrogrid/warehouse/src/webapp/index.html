AstroGrid data warehouse: front-end service

This web-application holds the front-end web-service for the AstroGrid Grid Data Warehouse; it is a facade through which clients can access the warehouse without dealing directly with Grid protocols. The warehouse web-service currently has exactly the same interface as AstroGrid's data-centre component.

The warehouse service is a SOAP service built on Apache-Axis, so it must be driven by a dedicated SOAP client (the classes in AstroGrid's data-centre component are suitable) rather than through these web pages. However, Axis provides the following administrative functions via your web browser.

  • Validate the local installation's configuration
    see below if this does not work.
  • View the list of deployed Web services
  • Call a local endpoint that list's the caller's http headers (or see its WSDL).
  • Visit the Apache-Axis Home Page
  • Administer Axis
    [disabled by default for security reasons]
  • SOAPMonitor
    [disabled by default for security reasons]

To enable the disabled features, the maintainer of this service should uncomment the appropriate declarations in WEB-INF/web.xml in the web-application and restart it.

Validating Axis

If the "happyaxis" validation page displays an exception instead of a status page, the likely cause is that the server of the web-application has multiple XML parsers in its classpath.

If you have problems getting Axis to work, consult the Axis Wiki and then try the Axis user mailing list.

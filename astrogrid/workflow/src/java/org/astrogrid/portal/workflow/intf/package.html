Public interface to the workflow system

Replaces classes in the {@link org.astrogrid.portal.workflow.jes} and {@link org.astrogrid.portal.workflow.design} packages.

Workflow Manager

Create a {@link org.astrogrid.portal.workflow.intf.WorkflowManagerFactory}, use this to create a {@link org.astrogrid.portal.workflow.intf.WorkflowManager}. This object provides components to
  • Create new workflow objects - {@link org.astrogrid.portal.workflow.intf.WorkflowBuilder}
  • Query application registy - {@link org.astrogrid.portal.workflow.intf.ApplicationRegistry}
  • Work with a myspace store - {@link org.astrogrid.portal.workflow.intf.WorkflowStore}
  • Submit and manage jobs - {@link org.astrogrid.portal.workflow.intf.JobExecutionService}

Activity Keys

@link org.astrogrid.portal.workflow.intf.ActivityKey} provides an implementation of activity keys based on the underlying JXPath methods of the object model. They can be used to record position in any object model whose objects extend {@link org.astrogrid.common.bean.BaseBean} (e.g. Workflow, Applicaton Descriptions).

Application Description

The {@link org.astrogrid.portal.workflow.intf.ApplicationDescription} class represents a single application. It provides methods that:
  • create and populate new {@link org.astrogrid.workflow.beans.v1.Tool} objects
  • query for descriptions of parameters (types, cardinality, documentation)
  • validate tool objects contain correct parameters for an application.

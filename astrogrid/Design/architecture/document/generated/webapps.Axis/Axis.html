The Axis-web-service pattern

AstroGrid web-applications including web services use the J2EE-servlet and Axis frameworks to connect SOAP messages with Java classes. The conceptual-parts diagram shows how these frameworks come together, and the physical-parts diagram shows the implementation chosen by Astrogrid.

AstroGrid adds to the normal usage of J2EE and Axis internal conventions for interconnecting the WSDL contracts and the Java code. These are shown in the service-interfaces diagram. This diagram also illustrates the use of delegate classes as a refinement of client-side stubs for the services.

Web applications following this pattern are modelled as specialized examples of the service-interfaces and physical-parts diagrams.

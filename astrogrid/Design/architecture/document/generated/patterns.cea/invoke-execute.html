package: patterns.cea

Invocation of execute on a CEA service

The execute operation starts a job-step previously set up by the init operation on the same service. The latter operation passes back an identifier for the job-step: this is the string argument to the execute method.

The job-step to be executed is stored in the service as an Application object. The execute operation is delegated from the client down to this Application. The Application returns as soon as the job-step is started; i.e. the logic of the step runs asynchronously to the execute operation. The Application returns true from execute() if the step was started successfully and false otherwise.

Class Diagram standard-generated-classes
package: patterns.cea

Following the Axis pattern, CEA service have a client stub and a service skeleton which both implement the service's interface. These classes are generated by tooling from the WSDL contract. Because all CEA service have the same WSDL contract, they have the same stub and skeleton.

There is also a "locator" class which is a concrete factory for the stubs. The client code sees the locator and the interface implemented by the objects that the locator creates; it does not see the details of the stubs. As with the stub and skeleton, the locator is generated from the WSDL contract and is the same for all CEA services.

The skeleton contains an object implementing the ExecutionController interface. The class of this object in a given service determines what the service does.

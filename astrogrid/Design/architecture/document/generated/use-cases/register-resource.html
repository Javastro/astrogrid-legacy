Activity Choose authority ID
User chooses the authority ID for the resource to be registered. In the current, AstroGrid system, this must be the single authority ID that is managed by the chosen registry. Use rcan look up this ID by querying the chosen registry through its web interface.

to Activity Choose resource key
Activity Choose registry
User chooses the publishing registry in which to make the registration. In a mature VO, all registrations eventually propgate to all full registries via the harvesting process, so the choice of publishing registry is a matter of convenience. In the short term, harvesting is probably incomplete, so User must choose the publishing registry to suit the intended spread of the information.

to Activity Choose authority ID
Activity Choose resource key
User chooses the resource key for the new resource. The key must be unique within the registry. User can trawl through the entire registry checking uniqueness, or can agree a key with the registry operator.
Transition links
to Activity Create VOResource
Activity Create VOResource

User creates the resource metadata, an XML document of type VOresource, as a local file. User should validate this file using XML tools; the registry does not validate the VOResources it receives.

Most registrations use a sub-type of the basic VOResource. The types that are most relevant to AstroGrid are:

ceaApplicationType
an application that can be run in a job-step
ceaServiceType
a job-manager service that can execute job-steps
serviceType
a generic, non-CEA service; used to register MySpace services/dd>
tabularSkyServiceType
a catalogue-archive service with non-CEA interfaces

The types refer to W3C XML schemata published by IVOA.

Activity Upload VOResource
User navigates to the registry's web interface using a web browser. User navigates to the page for adding a registry entry: this is a form with a text-entry box for entering a VODescription element. User copies his VOResource element into the desktop clipboard and pastes it into the VODescription in the form. user submits the form; the registry adds the VOResources to its collection of metadata.

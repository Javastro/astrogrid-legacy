Execution of an aplication by delegation to an HTTP service

In this collaboration, an application is run by calling an HTTP CGI service. This form does not apply if the application is delegated to a SOAP service.

The CEA web-service calls the HttpApplication. The latter uses HttpParameter adapters, set up when the HttpApplication was constructed, to build the name-value pairs which represent the arguments in a CGI call.

The HttpApplication constructs a HttpServiceClient. Constructing this object causes the HTTP call to the CGI service.

The HttpApplication spawns a thread to wait for the CGI call to complete, and then returns control the the web service, allowing the web-service operation to complete.

The Thread calls run() on the HttpApplication when it starts. This causes the HttpApplication to block waiting for the CGI call to complete.

When the CGI call returns, the HttpApplication writes the results into a HttpParameterAdapter. This will deliver them to MySpace if the parameter is mapped to a MySpace file.

Note:
Repeat for all ParameterAdapters

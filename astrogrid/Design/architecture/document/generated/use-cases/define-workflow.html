Activity Detail
Activity add job-step
User uses the menu system on the workflow-construction page to add a job-step. The step must be added within a sequence or parallel structure that is itself inside the workflow. User can add new structures of these types if needed.
Transition links
to Activity Define tool for job-step
Activity Define tool for job-step

User chooses the tool to be run by the job-step from a menu. "Tool" in this context means the business logic to be run for the job-step; the tool can be an archive query or an execution of a data-transforming application. In either case, user is selecting a web service to execute the job-step. In the current system, a specific service-endpoint is chosen when the workflow is designed. In later systems, user may be able to specify a class of tool, with the system choosing the endpoint from a set of equivalent services when the job-step is executed.

One of the possible tools is an archive query. In this case, the single, formal parameter is the location in MySpace of a file defining the query. User is assumed to have created this file previously.

Activity Locate workflow in MySpace
User presses the "load" button and uses the MySpace mini-browser (a pop-up window generated by the portal) to choose a workflow stored in MySpace. User can either either choose a workflow stored earlier by himself, or one from a library of workflows in a shared area of MySpace.

to Activity Return file contents
Activity Navigate to workflow page
User navigates, using a web browser, to the page for workflow construction. System displays a new workflow initialized with a sequence structure.

Activity Record workflow
User saves the file. Portal displays the MySpace microbrowser forcing User to enter an AstroGrid Storage Locator (AGSL). Portal send the XML defintiion of the workflow to the MySpace manager named in the AGSL.
Transition links
to Activity Update file contents
Activity Return file contents
Transition links
to Activity add job-step
Activity Set arguments for tool

Portal displays a web form generated from the formal parameters for the tool. User fills in this form with actual arguments for the job-step.

Arguments can be primitive objects (numbers, strings, boolean values) or names of files. In the latter case, User fills in references into MySpace using the MySpace microbrowser. The references are encoded as AstroGrid Storage Locators (AGSLs).

Activity Supply formal parameters
Registry supplies the description of the tool from the ceaApplication element of its registration. This structure states the tool's formal parameters.

Activity Update file contents
MySpace Manager receives the XML definition of the workflow from Portal together with an AGSL that includes the desired location in MySpace. MySpace manager locates the physical file corresponding to the AGSL, creating it if necessary. MySpace manager writes the data into the file.

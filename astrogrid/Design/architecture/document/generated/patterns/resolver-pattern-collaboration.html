Collaboration Diagram resolver-pattern-collaboration
package: patterns
A delegate object is created to give access to a specific web-service end-point, specified by an IVORN. The resource registry is invoked to determine the URL for this service endpoint: i.e. the registry resolves the IVORN to the URL. The resolver class hides the registry operation from the client code. The resolver follows the 'concrete fractory' pattern in that it creates a single, specific class of delegate. The delegate works a single, specific class of web service.

Note:
These vary with the implementation of the pattern, but the registry classes are always the same.
